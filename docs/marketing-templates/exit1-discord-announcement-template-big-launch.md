# Big Launch Discord Announcement Template

Use this for larger product launches, major features, new check types, new infrastructure, or big UX/product changes.

```md
# :rocket: Update

**Big one today.**

I’ve shipped **[feature/update name]**.

The goal is simple: **[one sentence explaining the product value]**.

## New

• **[Main feature]** — [What it does]  
• **[Second feature]** — [What it does]  
• **[Third feature]** — [What it does]

## Why this matters

[2 to 4 short sentences.]

Before this, [old pain/problem].  
Now, [new better state].  
That means [practical user benefit].

## Free for all

• [Free feature 1]  
• [Free feature 2]  
• [Free feature 3]

## Paid plans

• **Nano** — [What Nano gets]  
• **Pro** — [What Pro gets]  
• **Agency** — [What Agency gets]

## Behind the scenes

• [Infrastructure improvement]  
• [Reliability improvement]  
• [Performance improvement]  
• [Monitoring/alerting improvement]

I’m still tuning this, so please tell me if you see anything strange.

Especially false alerts, missed alerts, confusing UI, or anything that feels slower than it should.

As always, feedback is welcome.
```

## Notes

Best used when:

• The update deserves excitement  
• You shipped something users can immediately try  
• The feature changes how people use exit1  
• You want to explain the “why” without writing a blog post

This format works well for things like Live page updates, new monitoring types, API/MCP improvements, alerting upgrades, or major status page changes.
