# Exit1 Discord Announcement Templates

A small reusable set of Discord announcement templates for future exit1 updates.

The tone should stay:

• Founder-led  
• Clear  
• Practical  
• Slightly casual  
• Honest about tradeoffs  
• Focused on what changed and why it matters

---

# Discord Announcement Template

Use this for normal product updates, feature releases, fixes, and general progress announcements.

```md
# :mega: Update

**[Short punchy summary of the update]**

I’ve shipped a new update to exit1.

This one is mostly about **[main theme: reliability / speed / alerts / status pages / API / UI polish / monitoring depth]**.

## What’s new

• **[Feature 1]** — [Short explanation of what it does and why it matters]  
• **[Feature 2]** — [Short explanation of what changed]  
• **[Feature 3]** — [Short explanation of the user benefit]  
• **[Feature 4]** — [Optional, only include if meaningful]

## Available for

• Free: [What Free users get, or “No changes”]  
• Nano: [What Nano users get]  
• Pro: [What Pro users get]  
• Agency: [What Agency users get]

## Improvements

• [Performance / stability / reliability improvement]  
• [UI / UX improvement]  
• [False positive / alerting / monitoring accuracy improvement]  
• [Infrastructure improvement, if relevant]

## Fixes

• [Bug fix 1]  
• [Bug fix 2]  
• [Bug fix 3]

If you notice anything weird, false positives, false negatives, or something that feels off, please reach out.

I really want exit1 to stay fast, reliable, and simple to use.

As always, feedback is welcome.
```

## Notes

Best used when:

• You shipped multiple smaller things  
• You changed limits or plan access  
• You want a clean Discord update without over-explaining  
• The update is meaningful, but not a major launch

Keep it short. Discord announcements should feel alive, not like release notes.

# Big Launch Discord Announcement Template

Use this for larger product launches, major features, new check types, new infrastructure, or big UX/product changes.

```md
# :rocket: Update

**Big one today.**

I’ve shipped **[feature/update name]**.

The goal is simple: **[one sentence explaining the product value]**.

## New

• **[Main feature]** — [What it does]  
• **[Second feature]** — [What it does]  
• **[Third feature]** — [What it does]

## Why this matters

[2 to 4 short sentences.]

Before this, [old pain/problem].  
Now, [new better state].  
That means [practical user benefit].

## Free for all

• [Free feature 1]  
• [Free feature 2]  
• [Free feature 3]

## Paid plans

• **Nano** — [What Nano gets]  
• **Pro** — [What Pro gets]  
• **Agency** — [What Agency gets]

## Behind the scenes

• [Infrastructure improvement]  
• [Reliability improvement]  
• [Performance improvement]  
• [Monitoring/alerting improvement]

I’m still tuning this, so please tell me if you see anything strange.

Especially false alerts, missed alerts, confusing UI, or anything that feels slower than it should.

As always, feedback is welcome.
```

## Notes

Best used when:

• The update deserves excitement  
• You shipped something users can immediately try  
• The feature changes how people use exit1  
• You want to explain the “why” without writing a blog post

This format works well for things like Live page updates, new monitoring types, API/MCP improvements, alerting upgrades, or major status page changes.

# Plan Change Discord Announcement Template

Use this for pricing changes, tier changes, grandfathering, plan migrations, usage limits, or paid plan announcements.

```md
# :yellow_heart: Update to the plans

First of all, thank you for supporting exit1.

The product keeps getting more serious, and I want the plans to reflect that without making things confusing.

## What’s changing

• **[Plan/change 1]** — [Short explanation]  
• **[Plan/change 2]** — [Short explanation]  
• **[Plan/change 3]** — [Short explanation]

## Existing users

If you’re currently on **[old plan]**, you will be moved to **[new plan / grandfathered setup]**.

You keep **[important benefits they keep]**.

No action needed.

## Free users

Free is still available.

You still get:

• [Free benefit 1]  
• [Free benefit 2]  
• [Free benefit 3]

The main limit changing is:

• [Limit change, written clearly]

## Paid users

Paid plans now give you more room for serious monitoring:

• Faster checks  
• More history  
• More alerts  
• More status pages  
• More advanced monitoring features

I know plan changes are never the most exciting update, but they make it possible to keep building exit1 properly.

Thanks again for being here early.

As always, feedback is welcome.
```

## Notes

Best used when:

• You are changing pricing or limits  
• You need to reassure existing users  
• You are grandfathering users into better terms  
• You want to explain the business reason without sounding defensive

Be extra clear in this format. Pricing confusion creates more support work than almost anything else.

# Announcement Checklist

Before posting, check:

• Is the headline clear enough to understand in 3 seconds?  
• Did you say what changed?  
• Did you say who gets it?  
• Did you explain why it matters?  
• Did you mention anything users should report?  
• Did you avoid sounding too corporate?  
• Did you keep it short enough for Discord?

# Small Style Guide

Use:

• “I’ve shipped…” when it is clearly founder/product-led  
• “We’ve shipped…” when it feels more like a team/company announcement  
• “Free for all” when the update genuinely helps free users  
• “Behind the scenes” for infrastructure and reliability work  
• “As always, feedback is welcome.” as the consistent closing line

Avoid:

• Too much marketing language  
• Long paragraphs  
• Over-explaining infrastructure unless users benefit from it  
• Vague plan changes  
• Apologizing for paid features when the value is clear
